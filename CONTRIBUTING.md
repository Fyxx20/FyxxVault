# Contributing to FyxxVault

Thanks for your interest in contributing! FyxxVault is open source and welcomes contributions.

## How to contribute

1. **Fork** the repository
2. **Create a branch** for your feature (`git checkout -b feature/my-feature`)
3. **Make your changes** following the existing code style
4. **Test** your changes (`npm run build` in `web/`)
5. **Commit** with a clear message
6. **Push** and open a **Pull Request**

## Guidelines

- Follow existing code patterns (Svelte 5 runes, TypeScript, Tailwind CSS)
- Do not introduce new dependencies without discussion
- Never commit secrets, API keys, or credentials
- Security-sensitive changes require extra review

## Security

If you find a security vulnerability, please **do not** open a public issue. Instead, follow the [responsible disclosure process](SECURITY.md).

## Code of Conduct

Be respectful. We're all here to build something great.
